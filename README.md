# Pet Medication Tracker

> Мобильно-дружественное веб-приложение для отслеживания приёма лекарств у домашних животных

---

## 🎯 Описание проекта

Помогает владельцам животных не пропускать приём препаратов. Препараты сначала заводятся в личном каталоге (название, для чего, дозировка в одной таблетке — например «Энроксил 15 мг»), а затем применяются к питомцу как курс лечения: выбор препарата из каталога, дозировка за один приём, частота, время напоминаний, даты начала и окончания — система автоматически генерирует расписание доз на весь курс. Каждую дозу можно отметить как «принято», «пропущено» или «не принято» прямо из дневного календаря; просроченные дозы планировщик помечает автоматически, а пропущенная или непринятая доза не «теряется» — вместо неё дальше по графику появляется новая, пока курс не будет отменён вручную. Курс лечения можно в любой момент отменить (дозы после дня отмены стираются из календаря) или удалить (уже отмеченные дозы остаются в календаре, сам курс исчезает из списка). Препарат из каталога нельзя удалить, пока он используется хотя бы в одном активном курсе; после удаления история его приёмов в календаре сохраняется. По каждому завершённому курсу доступна сводная статистика приёма. При создании курса форма для таблеток/жидкости сама подсказывает, сколько препарата понадобится на весь курс, а в карточке отметки приёма дозировка подписана понятно («Дать 0.05 · 23:00»). Вкладка «Календарь» открывается мини-календарём на месяц с инициалами питомцев по дням; авторизация поддерживает восстановление пароля по email.

Целевая аудитория — владельцы домашних животных, которым нужно вести регулярный курс лечения (антибиотики, хронические препараты, витамины) без риска забыть дозу.

## 🚀 Быстрый старт

### Установка (Docker)

```bash
# Клонирование
git clone https://github.com/AlenaSygrysheva/pets_medication_tracker.git
cd pets_medication_tracker

# Конфигурация
cp .env.example .env
# Обязательно смените SECRET_KEY в .env

# Запуск (поднимет app + PostgreSQL + Redis, применит миграции)
docker compose up -d
```

Сервисы после запуска:

| URL | Описание |
|---|---|
| http://localhost:8000 | Веб-интерфейс (SPA) |
| http://localhost:8000/docs | Swagger UI |
| http://localhost:8000/redoc | ReDoc |
| http://localhost:8000/health | Health check |

### Локальная разработка (без контейнера с приложением)

Требования: Python 3.12+, [Poetry](https://python-poetry.org/docs/#installation) 1.8+, PostgreSQL 15 и Redis 7 (или только их поднять через Docker).

```bash
# Установка зависимостей
poetry install

# Поднять только БД и Redis
docker compose up -d db redis

# Применить миграции
poetry run alembic upgrade head

# Запустить сервер с автоперезагрузкой
poetry run uvicorn app.main:app --reload
```

### Конфигурация

Файл `.env` (скопировать из `.env.example`):

```env
# Обязательные
SECRET_KEY=your-super-secret-key-change-in-production

# Опциональные

ACCESS_TOKEN_EXPIRE_MINUTES=30
REFRESH_TOKEN_EXPIRE_DAYS=7

# Восстановление пароля (опционально)
```

Если `SMTP_HOST` не задан, письмо восстановления пароля не отправляется, а его содержимое (включая ссылку) пишется в лог приложения — удобно для локальной разработки без почтового сервера.

## 📚 API документация

После запуска доступны Swagger UI (`/docs`) и ReDoc (`/redoc`). Все эндпоинты требуют JWT, кроме `/auth/register`, `/auth/login`, `/auth/refresh`, `/auth/forgot-password`, `/auth/reset-password` и `/health`.

### Аутентификация

| Метод | Путь | Описание |
|---|---|---|
| `POST` | `/api/v1/auth/register` | Регистрация, возвращает токены |
| `POST` | `/api/v1/auth/login` | Вход, возвращает токены |
| `POST` | `/api/v1/auth/refresh` | Обновление access-токена |
| `GET` | `/api/v1/auth/me` | Данные текущего пользователя |
| `POST` | `/api/v1/auth/forgot-password` | Запросить ссылку для смены пароля на email (ответ одинаков независимо от того, зарегистрирован ли email) |
| `POST` | `/api/v1/auth/reset-password` | Сменить пароль по одноразовому токену из письма |

### Питомцы

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/api/v1/pets` | Список питомцев пользователя |
| `POST` | `/api/v1/pets` | Добавить питомца |
| `GET` | `/api/v1/pets/{id}` | Карточка питомца |
| `PATCH` | `/api/v1/pets/{id}` | Редактировать питомца |
| `POST` | `/api/v1/pets/{id}/avatar` | Загрузить фото |
| `DELETE` | `/api/v1/pets/{id}` | Удалить питомца |

### Каталог препаратов

Препарат — это справочная запись (название, «для чего», дозировка в одной таблетке), независимая от питомцев. Каталог у каждого пользователя свой. Одинаковое название с разной дозировкой — разные записи («Энроксил 15 мг» и «Энроксил 25 мг»); полный дубликат название+дозировка запрещён (`409`).

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/api/v1/drugs` | Каталог препаратов пользователя (без удалённых) |
| `POST` | `/api/v1/drugs` | Добавить препарат в каталог |
| `GET` | `/api/v1/drugs/{id}` | Карточка препарата |
| `PATCH` | `/api/v1/drugs/{id}` | Редактировать препарат |
| `DELETE` | `/api/v1/drugs/{id}` | Удалить препарат из каталога (мягко). Запрещено (`409`), если препарат используется хотя бы в одном активном курсе; история приёмов в календаре сохраняется в любом случае |

### Курсы лечения

Курс — это применение препарата из каталога к конкретному питомцу: дозировка за один приём, частота, время напоминаний, даты начала/окончания.

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/api/v1/medications/pet/{pet_id}` | Курсы лечения питомца (без удалённых) |
| `GET` | `/api/v1/medications/pet/{pet_id}/stats` | Сводка по завершённым курсам (отменённым или без оставшихся ожидающих доз) |
| `POST` | `/api/v1/medications` | Создать курс по `drug_id` (генерирует дозы по `reminder_times`) |
| `GET` | `/api/v1/medications/{id}` | Карточка курса (включает вложенный объект `drug`) |
| `PATCH` | `/api/v1/medications/{id}` | Редактировать курс |
| `POST` | `/api/v1/medications/{id}/cancel?as_of_date=` | Отменить курс на указанную дату (по умолчанию — сегодня): дозы после неё стираются из календаря, дозы до и в этот день помечаются `cancelled` |
| `DELETE` | `/api/v1/medications/{id}` | Мягкое удаление курса: пропадает из списка, но уже отмеченные дозы (`taken`/`skipped`/`missed`) остаются в календаре |

Пропущенная (`missed`) или непринятая вручную доза не сокращает курс — вместо неё автоматически создаётся новая доза дальше по графику (`MedicationService.extend_after_unresolved_dose`), пока курс не отменят явно. Поэтому курс считается «завершённым» не по дате окончания, а по отсутствию ожидающих доз (или по отмене).

### Календарь и дозы

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/api/v1/calendar/month/{year}/{month}` | Мини-календарь на месяц: по каждому дню — питомцы (инициалы) с запланированными приёмами, по всем питомцам пользователя |
| `GET` | `/api/v1/calendar/pet/{pet_id}/{date}` | Расписание дня (YYYY-MM-DD) |
| `PATCH` | `/api/v1/doses/{id}` | Отметить дозу (taken / skipped / missed) |

### Статусы дозы

| Статус | Описание |
|---|---|
| `pending` | Запланирована, ещё не принята |
| `taken` | Принята |
| `skipped` | Пропущена вручную |
| `missed` | Просрочена (проставлена планировщиком) или отмечена как «не принято» |
| `cancelled` | Курс отменён — доза на день отмены или раньше него, оставлена для истории |

- Пропущенная или непринятая доза больше не сокращает курс — вместо неё автоматически создаётся новая доза дальше по графику, пока курс не отменят явно; «завершённость» курса в статистике теперь определяется отсутствием ожидающих доз, а не датой окончания
- Восстановление пароля по email («Забыли пароль?» на экране входа) — одноразовая ссылка со сроком действия, письмо логируется вместо отправки, если SMTP не настроен
- Мягкое удаление курса лечения (`Medication.is_deleted`, миграция `005`) — курс пропадает из вкладки «Препараты», но уже отмеченные дозы остаются видны в календаре; раньше `DELETE` каскадно стирал курс и все его дозы без возможности восстановить историю
- Реализована отмена курса из карточки дозы («🛑 Отменить курс» в модалке рядом с «Принято» / «Пропустить» / «Не принято»): дозы после выбранного дня стираются из календаря, дозы до и в этот день помечаются статусом `cancelled` и остаются в истории
- Новая вкладка «Статистика приёма» и эндпоинт `GET /medications/pet/{pet_id}/stats` — сводка (принято/пропущено/не принято) по каждому завершённому курсу
- Настраиваемое время напоминаний (`reminder_times`) — раньше время доз генерировалось всегда с 8:00 с равным интервалом, теперь пользователь задаёт время каждой дозы сам
- Понятное сообщение об ошибке при вводе имени пользователя вместо email на экране входа (раньше форма молча ничего не показывала), а также при других ошибках формы входа/регистрации
